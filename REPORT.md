# Character Animation Plugin — Project Report

**Course:** Character Animation  
**Platform:** N8RO Simulation — C++ Plugin  
**Language:** C++17 | Windows x64

---

## 1. Overview

This project implements a procedural character animation controller as a C++ DLL plugin for the N8RO simulation platform. The plugin drives a humanoid character through four motion states — Stand, Walk, Run, and Jump — using mathematically generated joint angles computed in real time. No pre-recorded animation clips or motion capture data are used. All motion is produced through sine-wave functions, a state machine, and exponential joint interpolation evaluated every frame.

---

## 2. Objectives

- Generate all character motion procedurally through code
- Implement a state machine managing four distinct animation states
- Control 10 major joints of the humanoid skeleton per frame
- Respond to real-time keyboard input with immediate state transitions
- Interpolate smoothly between poses to eliminate visible snapping

---

## 3. System Architecture

The plugin is structured around four subsystems that execute sequentially each frame.

**Input Handler** reads keyboard state using `GetAsyncKeyState` and maps keys 1–4 to the four animation states. State changes are recorded with a timestamp to support time-based animations.

**State Machine** holds the current active state as an enum value (`STAND`, `WALK`, `FAST_WALK`, `JUMP`). The only automatic transition is Jump returning to Stand after 0.8 seconds.

**Animation Generator** computes a target angle triplet `(x, y, z)` in radians for each of the 10 controlled joints based on the active state and simulation time.

**Interpolation and Output** blends current joint angles toward their targets using exponential smoothing and writes the results to the engine's joint override list each frame.

---

## 4. Motion States

### Stand
All leg joints held at zero rotation. Shoulder joints rotated −1.57 rad on the Z-axis to bring arms to the character's sides, correcting the default T-pose rest position.

### Walk
Sine-wave driven at 6.0 rad/s with leg swing amplitude of 0.40 rad and arm swing of 0.30 rad. Left and right hips are 180° out of phase. Arm swing is contra-lateral — left arm swings forward when right leg swings forward. Knee flexion scales with hip swing magnitude.

### Run
Identical structure to Walk with frequency doubled to 12.0 rad/s, leg amplitude increased to 0.70 rad, and arm amplitude to 0.60 rad. Elbow bend is introduced, scaling with arm swing velocity.

### Jump
A static split pose held for 0.8 seconds: left hip at +0.8 rad, right hip at −0.8 rad, right knee bent at 0.5 rad, both shoulders raised. After 0.8 seconds the state machine automatically returns to Stand.

## 5. Controlled Joints

| # | Joint | Role |
|---|-------|------|
| 1 | `leftHip` | Forward/backward swing of left leg |
| 2 | `rightHip` | Forward/backward swing of right leg (opposite phase) |
| 3 | `leftKnee` | Flexion during back phase of left leg swing |
| 4 | `rightKnee` | Flexion during back phase of right leg swing |
| 5 | `leftAnkle` | Ground contact orientation, left foot |
| 6 | `rightAnkle` | Ground contact orientation, right foot |
| 7 | `leftShoulder` | Arm swing and side-down positioning |
| 8 | `rightShoulder` | Arm swing opposite to left, mirrored Z offset |
| 9 | `leftElbow` | Elbow bend during run and jump states |
| 10 | `rightElbow` | Mirrors left elbow bend |

## 6. Animation Mathematics

Locomotion is generated by the formula:

```
joint_angle = sin(t × frequency) × amplitude
```

Where `t` is simulation time in seconds, `frequency` controls cycle speed, and `amplitude` controls range of motion.

Bilateral symmetry is achieved by negating the value for the opposite side. Knee flexion is derived from the hip value: `abs(hip_swing) × scale`, coupling knee and hip without an independent oscillator.

Smooth transitions use exponential smoothing each frame:

```
alpha  = 1 − exp(−15 × dt)
current += (target − current) × alpha
```

This converges quickly when the difference is large and slows as values approach, producing a natural ease-out without explicit curves.

## 7. Challenges and Solutions

**T-Pose correction** — The skeleton's zero-rotation rest pose is a T-pose. Corrected by applying a −1.57 rad Z-axis offset to both shoulders in all states.

**Joint axis calibration** — Each joint's X/Y/Z axes do not align with global directions. Correct axes were found by applying test rotations and observing limb direction in the viewer.

**Arm phase** — Initial implementation swung the arm in phase with the leg on the same side, which is biomechanically incorrect. Negating the shoulder value relative to the same-side hip corrected this to natural contra-lateral motion.

**Interpolation at zero dt** — When the engine evaluates multiple frames at the same timestamp, dt becomes zero and the interpolation stalls. Fixed by clamping dt to a minimum of 16 ms when it falls below zero or exceeds one second.

---

## 8. Results

The plugin loads correctly into N8RO and drives the character through all four motion states in response to keyboard input. Walk and Run produce visually convincing bipedal gait with correct leg alternation and contra-lateral arm swing. The Jump state presents a clear dynamic pose before returning to Stand automatically. All state transitions are smooth with no visible discontinuities. Performance is well within real-time requirements as all computation is limited to floating-point arithmetic and map lookups.

---

## 9. Conclusion

This project demonstrates that procedural animation using sine-wave joint control, a simple state machine, and exponential interpolation is sufficient to produce convincing humanoid motion in real time without any animation asset dependencies. The development process provided practical experience in joint axis calibration, biomechanically correct gait construction, and the integration of a C++ plugin into a live simulation engine. The modular architecture of the codebase supports straightforward extension with additional motion states or more advanced blending in future work.
